洛溪xx2024-09-022024-09-03

什么是 Hexo?

Hexo 是一个快速，简单，强大的博客框架。 它使用 Markdown (或者其他的渲染引擎)转换你的文章并结合一个漂亮的主题生成静态文件，而这一切都在数秒之内完成

Hexo 的优点

不需要服务器,一条指令即可快速部署

主题插件丰富,免费使用

直接生成静态网站，轻便、稳定、快速

手动安装直接进入程序官网安装就行,安装 Node.js建议选择 LTS 版官网下载

此步骤对于上面的系统应该都是一样的步骤，在你想要存放本地 Blog 源文件的目录打开 Terminal/Bash

hexo常用命令

本地预览

在 Blog 目录打开 Terminal/Bash 使用命令hexo g

生成静态页面后再执行hexo s启动本地预览,默认端口号是4000

指定端口号的命令为hexo s -p 端口

此时可以打开浏览器输入localhost:端口访问网站,按下ctrl + c可取消本地预览

这只是本地预览,要想让别人能访问你的网站,还需要完成部署流程

这里我用的 Github Pages 部署网站,你也可以使用其他的

Github 部署

打开github官网,没有账号可以去注册

新建一个仓库,仓库名填写为username.github.io
username 是你的 GitHub 用户名,仓库可见性设为public(否则别人访问不了)

连接本地 Git 和 GitHub

打开本地 Terminal/Bash

复制输出的密钥文件路径,使用文本编辑器打开id_rsa.pub文件,把内容复制下来

打开GitHub,点击头像的 settings 选项进入设置，在 SSH and GPG keys 中粘贴 id_rsa.pub 内容，并点击 Add SSH key

回到本地的 Terminal/Bash，输入命令连接 GitHub

出现一个 hostname 提示输入 yes 回车,看到这个就是连接成功的

修改 Blog 目录下的 Hexo 全局配置文件 _config.yml 的 deploy 字段

再在 blog 目录下安装一个插件，来完成推送到 github 的事:

hexo三连指令推送hexo clean & hexo g & hexo d

现在你的博客应该推送到 GitHub 仓库了,如下图所示打开 GitHub 仓库的 Pages 服务,注意对应仓库的分支和路径!!

Github绑定自定义域名

默认域名是你的用户名.github.io

虽然免费,但是没法被搜索引擎收录,绑定自己的域名会更好

在博客根目录的source目录下创建一个名字为CNAME的文件(该文件没有后缀!全部大写),里面写上你想要绑定的域名

使用hexo三连指令推送更新到仓库,在域名的 DNS 解析记录增加一条 CNAME 解析到 username.github.io. 的记录

等解析记录生效后,去 GitHub 仓库的 Settings 的 Pages 字段设置你的自定义域名并强制启用 HTTPS，现在你的博客应该可以通过 https://your.domain 访问了
